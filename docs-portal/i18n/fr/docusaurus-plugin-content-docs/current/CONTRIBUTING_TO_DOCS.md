# Contribuer à la documentation PrimeCalendar {#contributing-to-primecalendar-documentation}

## Processus {#process}

1. Branche à partir de `main`.
2. Mettez à jour les documents canoniques sous `docs/`.
3. Exécutez des contrôles de qualité des documents.
4. Soumettez le PR avec le contexte et les notes de vérification.

## Critères d'examen {#review-criteria}

- Précision technique
- Conformité au guide de style
- Intégrité du lien
- exhaustivité des métadonnées

## Problèmes de signalement {#reporting-issues}

- Ouvrez un problème GitHub avec l'étiquette `documentation`.
- Incluez le chemin, le problème et le comportement attendu.
- Joignez des captures d'écran pour les problèmes de documentation liés à l'interface utilisateur.

## Entretien trimestriel {#quarterly-maintenance}

- Réexécutez l’inventaire des documents et l’analyse des écarts.
- Archivez les pages obsolètes.
- Actualisez les captures d'écran et les exemples.
- Mettre à jour les KPI et les entrées du tableau de bord de santé.
