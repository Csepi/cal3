# Beitrag zur PrimeCalendar-Dokumentation {#contributing-to-primecalendar-documentation}

## Prozess {#process}

1. Zweig von `main`.
2. Aktualisieren Sie kanonische Dokumente unter `docs/`.
3. Führen Sie Qualitätsprüfungen für Dokumente durch.
4. Senden Sie PR mit Kontext- und Verifizierungshinweisen.

## Überprüfungskriterien {#review-criteria}

- Technische Genauigkeit
- Einhaltung von Styleguides
- Linkintegrität
- Vollständigkeit der Metadaten

## Probleme melden {#reporting-issues}

- Öffnen Sie ein GitHub-Problem mit der Bezeichnung `documentation`.
- Geben Sie Pfad, Problem und erwartetes Verhalten an.
- Hängen Sie Screenshots für UI-bezogene Dokumentprobleme an.

## Vierteljährliche Wartung {#quarterly-maintenance}

- Führen Sie die Dokumenteninventur und Lückenanalyse erneut durch.
- Archivieren Sie veraltete Seiten.
- Aktualisieren Sie Screenshots und Beispiele.
- Aktualisieren Sie KPIs und Gesundheits-Dashboard-Eingaben.
