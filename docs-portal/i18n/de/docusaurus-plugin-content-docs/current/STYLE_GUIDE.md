# PrimeCalendar Dokumentations-Styleguide {#primecalendar-documentation-style-guide}

## Schreibstandards {#writing-standards}

- Verwenden Sie die zweite Person und die aktive Stimme.
- Definieren Sie Fachbegriffe bei der ersten Verwendung.
- Halten Sie Sätze kurz und direkt.
- Bevorzugen Sie explizite Aktionsverben.

## Strukturanforderungen {#structure-requirements}

- Fügen Sie Metadaten-Frontmatter auf allen kanonischen Seiten hinzu.
- Befolgen Sie die Reihenfolge der Vorlagenabschnitte.
- Beziehen Sie Fehlerbehebungs- und zugehörige Ressourcen mit ein.
- Fügen Sie Beschriftungen zur Bearbeitungszeit und zum Schwierigkeitsgrad hinzu.

## Visuelle Standards {#visual-standards}

- Speichern Sie Assets in `docs/assets/`.
- Fügen Sie Alt-Text für alle Bilder hinzu.
- Verwenden Sie einen einheitlichen Anmerkungsstil.
- Aktualisieren Sie Screenshots, wenn sich die Benutzeroberfläche ändert.

## Barrierefreiheit {#accessibility}

- Behalten Sie die Reihenfolge der semantischen Überschriften bei.
- Verwenden Sie beschreibende Links (vermeiden Sie „hier klicken“).
- Sorgen Sie für kontrastreiche Bilder.
- Stellen Sie Transkripte für Videolinks bereit.

## Qualitätskontrollen {#quality-checks}

- Validieren Sie die Links vor dem Zusammenführen.
- Überprüfen Sie Rechtschreibung und Grammatik.
- Bestätigen Sie die Version/zuletzt aktualisierte Metadaten.
- Überprüfen Sie Beispiele anhand des aktuellen Verhaltens.
